# ML AI Capstone Project

**Section 1: project overview**
This capstone project is designed to build and optimise an ML model within a simulated black-box environment, which means we see both inputs and outputs, but we dont have access to what's happening inside the model.

In keeping with this black-box setup, and to reflect real-world scenarios, we also don't have access to the exact evaluation metrics during this project. Instead, the focus is to make iterative, evidence-based improvements to optimisation model's performance.

Understanding how to optimise a black-box use case can help in building strong intuition for decision-making under uncertainty, balancing exploration and exploitation, and iteratively refining models, which is a key skill for a data scientist when faced with incomplete information. It reinforces probabilistic thinking, disciplined experimentation, and model / strategy refinement as evidence accumulates.

**Section 2: DATA : inputs and outputs**
We are working with 8 synthetic black-box functions of increasing dimentionality from 2D to 8D. These are unknown mathematical functions that accept inputs and return a single output. The goal is to find the inputs that give the highest possible output for each function.

Every function F1-F8 is a maximisation problem. We won't see the internal workings of the functions, but we will observe how they respond to different new query data points (inputs).
The functions F1 and F2 are 2D arrays, F3 is a 3D array, F4 and F5 are 4D arrays, F6 is a 5D array, F7 is a 6D array and F8 is a 8D array. The output of all functions F1-F8 is a 1D array.
Each new query point (inputs) for functions F1-F8 follows format: x1-x2-x3-...-xn, where each xᵢ begins with 0 and is specified to six decimal places decimal places (for example, 0.123456-0.654321 for a 2D function). 

At each iteration of the submission, the new input query point (generated by the optimisation model) and the respective generated output point (provided by the course leads) is appended to the F1-F8 input data files, to be used for the subsequent iteration.

**Section 3: Objective**

The ultimate objective of the back-box optimisation project is to find the inputs that give the highest possible output for each function. Every function is a maximisation problem.

The project works in an iterative manner, with every iteration providing new query points to generate outputs.


**Section 4: Technical details - Model and Hyperparameter optimisation**
The approach to generate query points is refined as through a better understanding of the Gausian process optimisation, acquisition functions and degree of exploration vs exploitation is obtained.

**1st Iteration: Initial methodology**
The query points were selected using the Expected Improvement criterion, which balances exploitation and exploration. Exploitation focuses on sampling regions where the model predicts high output values based on current observations, while exploration targets regions with high uncertainty where the model is less confident. This balance allows the algorithm to efficiently improve performance without becoming stuck in local optima or wasting samples in uninformative regions.

**2nd Iteration: Assessing EI and UCB acquisition approaches and balancing exploitation vs exploration**
The 2nd strategy was consensus-based selection: new query points were chosen when both Expected Improvement (EI) and Upper/Lower Confidence Bound (UCB/LCB) proposed nearly the same location. This was prompted primarily by model-driven signals, not heuristics on the input space. Agreement between EI and UCB indicates that the surrogate model’s predicted mean and uncertainty structure were aligned, suggesting a high-confidence region where both exploitation-focused and exploration-aware criteria agreed.

**3rd Iteration: Assessing TS and UCB acquisition approaches, with focus on exploration**
This round is focussed on more exploration (instead of a balanced exploration - exploitation approach). To identify better new query points in diverse, uncertain regions (exploration), Thompson Sampling (TS) approach and UCB were used to identify new query points. The result of TS approach was compared to the results of UCB approach to select new query points. Acquisition parameters (e.g., exploration weight) were adjusted heuristically to encourage stronger exploration.
